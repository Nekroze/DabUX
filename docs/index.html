features

  • General Usage - The main dab entrypoint is actually a small posix complient shell script that wraps docker and starts a container using the dab image for you. This script is designed to be lightweight and portable and should rarely be updated as the rest of the code for dab lives inside the image.
  • Subcommand: apps - Provides easy access to common/powerful apps.
  • Subcommand: config - The config subcommand manages the dab configuration key value store. Config keys are any word or sequence of words (for namespacing) delimited by a forward slash, eg. `foo/bar` would store the url for the dab repository.
  • Subcommand: group - The group subcommand manages groups of groups and repostory entrypoints.
  • Subcommand: network - The network subcommand provides management over the lab network.
  • Subcommand: repo - The repo subcommand manages configured git repositories.
  • Subcommand: shell - The shell subcommand gives access to the dab environment from which all scripts are executed, this is mostly for debug purposes.
